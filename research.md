### Бот 1. Protocol Digital SEO Bot

**Сайт:** https://protocol-digital.ru/seo-audit-bot/

**Какую задачу решает.** Владелец сайта или маркетолог хочет быстро понять, почему сайт плохо ранжируется, без найма SEO-специалиста и без разбора сложных инструментов вроде Ahrefs. Он просто кидает ссылку в Telegram и через 5 минут получает конкретный список проблем.

**Функции.**
Анализ по 18 параметрам: SEO (мета-теги, заголовки, структура), безопасность (HTTPS, уязвимости), Core Web Vitals (скорость загрузки, LCP, CLS). Отдельный блок «AI-оптимизация» — как сайт воспринимают языковые модели при генерации ответов (свежий угол под LLM-поисковики). На выходе — PDF-отчёт с оценками и приоритизированными рекомендациями. Есть 5 бесплатных аудитов в неделю, далее платная подписка. Бот умеет отдавать как короткую выжимку, так и полный документ.

**Типичный диалог.**
```
Пользователь: /start
Бот: Привет! Отправьте URL сайта для проверки.
Пользователь: myshop.ru
Бот: Анализирую, подождите около 5 минут...
Бот: Готово! Ваш сайт набрал 54/100. Критические ошибки:
     — отсутствует meta description на 12 страницах
     — скорость загрузки мобильной версии 8.2 сек (норма до 3 сек)
     — нет HTTPS на поддомене
     [PDF-отчёт прикреплён]
```

**Что делает его полезным.** Скорость и нулевой порог входа — не нужна регистрация, нет личного кабинета. PDF-отчёт сразу можно показать клиенту или отправить по почте. Агентство Protocol Digital использует бота как витрину: клиент видит проблемы → агентство предлагает их решить.

**Что взять.**
- PDF/HTML как осязаемый артефакт, который живёт за пределами Telegram и уходит наружу (виральность)
- Блок «AI-оптимизация» — готовность сайта к LLM-поисковикам как отдельный критерий
- Лимит 5 бесплатных аудитов — создаёт ощущение ценности без блокировки первого использования

### Бот 2. SEO-аудит бот с Habr (публичный кейс)

**Источник:** https://habr.com/ru/articles/996844/

**Какую задачу решает.** SEO-специалист или продвинутый владелец сайта хочет автоматизировать рутину: регулярные проверки страниц, поиск битых ссылок, мониторинг изменений. Инструмент профессиональнее Protocol Digital — для тех, кто знает, что такое краулинг и метатеги.

**Функции.**
Архитектура разбита на 8 модулей: SEO-аудит страницы по 40+ параметрам, краулер до 100 страниц с проверкой 500 ссылок, модуль «Баден-Баден» (11 признаков переоптимизации для Яндекса), мониторинг с алертами по расписанию, сравнение двух аудитов (diff до/после правок).

**Типичный диалог.**
```
[inline-меню] → кнопка «SEO-аудит страницы»
Бот: Введите URL страницы.
Пользователь: example.ru/product
Бот: Запустил проверку, обычно 30-60 сек...
Бот: Готово. Оценка: 68/100. Найдено:
     — нет canonical
     — 3 изображения без alt
     — title 87 символов (норма до 70)
     [Полный HTML-отчёт — кнопка]
```

**Что делает его полезным.** Глубина анализа на уровне платных SaaS, при этом всё прямо в Telegram без открытия браузера. HTML-отчёт с тёмной темой и структурированными секциями смотрится профессионально.

**Что взять.**
- HTML-отчёт с тёмной темой — профессионально, не требует ничего устанавливать
- **Сравнение двух аудитов** (до/после) — сильный инструмент для повторных продаж: клиент видит прогресс и возвращается
- Inline-кнопки вместо текстовых команд — снижает порог для нетехнических пользователей

### Инструмент 3. MyWebAudit (США, для агентств)

**Сайт:** https://www.mywebaudit.com/

**Какую задачу решает.** Это инструмент для агентства, а не для конечного клиента. Агентство встраивает виджет аудита к себе на сайт → посетитель вводит свой URL → получает отчёт → агентство получает контакт с болевой точкой.

**Функции.**
Три блока: SEO (мета-теги, заголовки, индексируемость), доступность (WCAG), производительность (Core Web Vitals). Отчёт генерируется в PDF с брендингом агентства — лого, цвета, контакты. Есть модуль «Better Leads» — по заданным критериям находит компании в интернете с конкретными проблемами на сайте, строит список потенциальных клиентов. От $49/мес для агентств.

**Типичный диалог (виджет на сайте агентства).**
```
[Блок на сайте] "Узнайте, что мешает вашему сайту продавать — бесплатный аудит за 60 секунд"
Посетитель вводит: URL сайта + email
[Через 60 сек на почту] PDF с оценками, проблемами, контактами агентства
[В PDF] кнопка "Хочу исправить"
[Агентство получает в CRM] email, URL, проблемы, "температуру" сайта
```

**Что делает его полезным.** Агентство получает не просто лид, а квалифицированный лид с болью. Продавец звонит: «Я вижу, что у вас скорость 9 секунд на мобильных — это съедает ~40% конверсий». Разговор начинается с конкретной проблемы, а не холодного питча. Результат: 80% закрытие сделок (vs 30% в среднем), средний чек $5,000+.

**Что взять.**
- **Брендированный отчёт** — документ с именем Александра внизу работает как визитка, которую клиент сохраняет
- **Разделение отчёта на две версии**: для клиента (проблемы простым языком) и для исполнителя (технические рекомендации) — MyWebAudit делает именно так
- Идея «температуры» сайта — единый показатель боли лида для исполнителя

### Инструмент 4. HubSpot Website Grader

**Сайт:** https://website.grader.com/

**Что анализируют.** 4 блока — Performance, Mobile, SEO, Security. Внутри: 17+ метрик. Результат — единый балл из 100.

**Как выглядит отчёт.** Один большой балл сверху (например, 43/100), затем декомпозиция по 4 блокам с отдельными баллами. По каждому — красные/жёлтые/зелёные флаги. Отчёт отправляется на email — так HubSpot захватывает лид.

**Ключевые механики конверсии.**
- Средний балл по 1 миллиону сайтов — 60.9/100 (оценка D−). Фраза «ваш сайт хуже среднего» — мощный триггер действия
- Email-захват до полного отчёта создаёт возможность follow-up
- 30 секунд на анализ — мгновенная ценность без регистрации

**Что взять.**
- Сравнение со средним по рынку («большинство сайтов набирают X»)
- Мгновенная ценность в первые 30 секунд

### Инструмент 5. SEOptimer

**Сайт:** https://www.seoptimer.com/

**Что анализируют.** SEO, Usability, Performance, Social, Security. Отдельно — наличие и качество CTA на странице. Более 100 проверок.

**Как выглядит отчёт.** Буквенные оценки A/B/C/D/F по каждому блоку (как школьные). Цветовая маркировка. Список рекомендаций с приоритетом.

**Ключевая механика монетизации.** SEOptimer продаёт встраиваемый виджет аудита агентствам за $59/мес. Более 2000 агентств используют эту схему. Белый лейбл — агентство выглядит как эксперт.

**Что взять.**
- Буквенные оценки A/B/C/D/F — психологически понятны всем без объяснений. F — провал, C — есть что улучшать, A — хочется показать
- Конкретный ущерб от проблемы, а не абстрактное «это важно»

## Часть 2. Проблемы существующих инструментов

### Техническая перегруженность

Главная жалоба пользователей — инструменты созданы для специалистов, не для владельцев бизнеса. Screaming Frog выдаёт сотни строк технических данных без объяснений. Semrush и Ahrefs имеют крутую кривую обучения. Один инструмент предлагает 40+ разделов — человек видит это и закрывает вкладку. Отчёты пишут «для SEO-специалиста» — с жаргоном и техническими деталями, которые отталкивают владельцев сайтов.

### Нeдействующие рекомендации

Отчёты фиксируют проблемы, но не дают приоритетов. Владелец малого бизнеса видит 87 ошибок и не понимает: что критично, что можно игнорировать, что делать первым. Инструменты говорят «у вас медленная страница» — но не объясняют, что это значит для продаж и как именно исправить без программиста.

### Цена отсекает аудиторию

Semrush — от $117/мес, Ahrefs — от $99/мес. Для малого бизнеса это неподъёмно за инструмент, в котором они всё равно ничего не поймут.

### Telegram-боты для аудита практически отсутствуют

Поиск показал: существуют боты для оптимизации самих Telegram-каналов, но полноценных ботов для аудита внешних сайтов — единицы, и они слабые. **Это незанятая ниша.**

## Часть 3. Что ИИ даёт, чего нет у технических инструментов

**Объяснение человеческим языком.** Технический инструмент пишет: «LCP 4.2s, CLS 0.15». ИИ пишет: «Ваш сайт грузится медленно — посетители уходят не дождавшись. Это напрямую снижает продажи». ИИ переводит код ошибок в бизнес-последствия.

**Анализ смысла, а не только структуры.** Технические инструменты не читают текст — они считают теги. ИИ понимает, насколько контент соответствует намерению пользователя, насколько убедителен заголовок, есть ли доверительные сигналы (E-E-A-T). AI-инструменты выявляют 85–90% критических проблем против 60–70% у ручного аудита.

**Приоритизация по бизнес-влиянию.** ИИ может выбрать: «вот 3 проблемы из 87, которые влияют на конверсию сильнее всего — исправьте их первыми». Технический crawler просто перечисляет всё подряд.

**Адаптивность под контекст.** ИИ учитывает нишу сайта: рекомендации для интернет-магазина отличаются от рекомендаций для лендинга услуг.

## Часть 4. UX-практики: как показывать оценки

**Формула: диагностика + надежда + путь** (не просто shame).

Правильный паттерн:
1. Показать реальный балл (без прикрас)
2. Сравнить со средним («большинство сайтов набирают X»)
3. Назвать 2-3 главных проблемы с понятным ущербом («медленная загрузка = потеря 30% посетителей»)
4. Показать, что будет после исправления («после оптимизации обычно +15-20 баллов»)

**Что обязательно включать в отчёт:**
- Единый балл — одно число, которое можно запомнить и сравнить
- Декомпозиция по категориям (не просто «плохо», а «плохо вот здесь»)
- Приоритизация проблем — что чинить первым
- Конкретный ущерб от проблемы, а не абстрактное «это важно»
- Сравнение с бенчмарком (средние по рынку)

**CTA с высокой конверсией — принцип:** описывает результат, а не действие.

| Слабо | Сильно |
|-------|--------|
| «Связаться с нами» | «Исправить эти 3 проблемы за одну неделю» |
| «Узнать больше» | «Получить план исправлений бесплатно» |
| «Отправить заявку» | «Показать отчёт разработчику → он скажет сколько стоит» |

## Часть 5. Рекомендации для доработки спецификации

### 1. «Переводчик с технического на человеческий» — главное преимущество

После каждого показателя — одно предложение на языке бизнеса. Пример:
> «Ваш сайт получил 43/100 по скорости. Это значит, что ~50% мобильных пользователей уходят до загрузки — вы теряете каждого второго посетителя с телефона.»

Никто из существующих инструментов не делает это автоматически в Telegram.

### 2. Топ-3 действия вместо списка из 80 проблем

Завершать отчёт разделом «С чего начать прямо сейчас: 3 шага». ИИ выбирает три изменения с максимальным эффектом при минимальных затратах. Снимает паралич от избытка информации.

### 3. Два отчёта — сильный дифференциатор

Нигде на рынке нет двух форматов в Telegram:
- **Для клиента:** «что болит и почему это важно» — простым языком, без жаргона
- **Для исполнителя:** технические детали + рекомендации + оценка трудоёмкости

Это прямой инструмент продаж для веб-разработчика.

### 4. Анализ контента, не только техники

PageSpeed даёт технику. Claude даёт смысл. Добавить оценку:
- Понятно ли что продаёт сайт за 5 секунд
- Есть ли призыв к действию выше сгиба
- Есть ли доверительные элементы (отзывы, гарантии, контакты)
- Убедителен ли заголовок

Это то, что не умеет ни один автоматический инструмент.

### 5. Оценка «готовности к продажам», а не «SEO-score»

Итоговый показатель не «SEO-score 67/100», а «Ваш сайт готов продавать на 4/10». С объяснением: что мешает конверсии, не только видимости в поиске. Владельцу малого бизнеса это понятнее и ближе к деньгам.

### 6. Буквенная оценка как якорь

Добавить к числовой оценке букву: A/B/C/D/F. F — интуитивно понятно, что это провал. A — хочется показать друзьям. Пример: `Общая оценка: 52/100 — C–`

### 7. Сравнение с конкурентом (расширенный режим)

Пользователь вводит URL конкурента → бот сравнивает:
> «Ваш конкурент грузится за 1.2 сек, ваш сайт — за 4.1 сек. Вы проигрываете по скорости в 3 раза.»

Достаточно одного конкурента. Контекст мотивирует сильнее, чем абстрактная цифра.

### 8. Мини-аудит за 30 секунд как точка входа

Перед полным отчётом — «экспресс-диагноз» в 3-4 строки: быстро/медленно, есть ли критические ошибки, общее впечатление. Полный отчёт — по кнопке «Хочу подробнее». Конверсия в полный аудит выше, когда человек уже видит первый результат.

### 9. Сравнение двух аудитов (до/после)

Хранить результаты аудитов в БД. При повторной проверке того же сайта — показывать динамику. Это создаёт продолжение отношений и мотивирует возвращаться.

### 10. Брендирование отчёта

Имя и контакт Александра внизу каждого отчёта — HTML-файл работает как визитка, которую клиент сохраняет, пересылает и показывает коллегам.

## Главный вывод

Рынок полон технических инструментов для специалистов. Telegram-ботов для аудита внешних сайтов — практически нет. Ниша «полный аудит на человеческом языке, бесплатно, прямо в мессенджере» не занята.

Ключевое преимущество нашего бота — не в технических метриках (их все умеют считать), а в способности Claude объяснять смысл этих метрик для человека без SEO-образования. Плюс два формата отчёта — нигде такого нет.

**Следующий шаг:** добавить в спецификацию блок «Анализ контента» (убедительность заголовка, наличие CTA выше сгиба, доверительные элементы) и переименовать итоговую метрику с «средней оценки» на «готовность сайта к продажам».
